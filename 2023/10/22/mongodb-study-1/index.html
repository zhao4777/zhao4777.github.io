3.2 文档查询

  • 案例引入

    已知脚本books.js(在appdb中)，若想查其中title为book-20的书该怎么查？

    #登入mongo shell后，进入appdb
db.books.find()
//出现脚本内容
db.books.find(&#123;title:"book-20"&#125;)
//出现book-20的属性内容

文档查询语法：

db.collection.find(query,projection)
例：db.books.find(&#123;type:"novel",tag:"developer"&#125;)
//返回type为novel，tag为developer的书的所有属性
例：db.books.find(&#123;type:"novel",tag:"developer"&#125;,&#123;title:1,author:1&#125;)
//&#123;type:"novel",tag:"developer"&#125;:查找type为novel和tag为developer的书；
//&#123;title:1,author:1&#125;：返回所查找的书的title和author


还可以只返回第一项

db.books.findOne(&#123;type:"novel",tag:"developer"&#125;,&#123;title:1,author:1&#125;)

  • 当要查询文档中某一项大于某数字的文档时

    当要查询books中收藏数(favCount)大于60的书时

    db.books.find(&#123;type:"travel",favCount:&#123;$gt:60&#125;&#125;)
//$gt:表示大于

**查找条件对照表**
![图片](图片/3.png)
**查询逻辑运算符**
  • $lt: 存在并小于
  • $lte: 存在并小于等于
  • $gt: 存在并大于
  • $gte: 存在并大于等于
  • $ne: 不存在或存在但不等于
  • $in: 存在并在指定数组中
  • $nin: 不存在或不在指定数组中
  • $or: 匹配两个或多个条件中的一个
  • $and: 匹配全部条件

查找逻辑对照表

例：db.books.find(&#123;title:&#123;$in:["book-1","book-2"]&#125;&#125;)
//查找title为book-1和book-2的书
#输出结果
//&#123; "_id" : ObjectId("63346fa3dd53a05e7a9084b9"), "title" : "book-1", "type" : "sociality", "tag" : "nosql", "favCount" : 28, "author" : "xxx1" &#125;
//&#123; "_id" : ObjectId("63346fa3dd53a05e7a9084ba"), "title" : "book-2", "type" : "novel", "tag" : "popular", "favCount" : 82, "author" : "xxx2" &#125;

降序查询

想要查询tag为nosql和developer的书，并将其已favCount的降序的方式打印出来

db.books.find(&#123;tag:&#123;$in:["nosql","developer"]&#125;&#125;).sort(&#123;favCount:-1&#125;)
//-1为降序

分页查询

db.books.find().skip(8).limit(4)
//表实将所查到的数据跳过前八行（即从第九行开始输出），并且只输出4行

正则表达式匹配查询

若想查找type里面有”so”字符串的书

db.books.find(&#123;type:&#123;$regex:"so"&#125;&#125;) 
//或者 
db.books.find(&#123;type:/so/&#125;)

3.3更新文档（2022.10.10-2022.10.21）

命令通用格式如下：

db.collection.update(query,update,options)
  • query：描述更新的查询条件
  • update：描述更新的动作及内容
  • options：描述更新的选项
    • upsert：若不存在update记录，是否插入新记录（默认为false，不插入）
    • multi：是否按条件查询出的多条记录全部更新（默认为false，只更新第一条）
    • writeConcern：决定一个写操作落到多少个节点上算成功
1. 更新操作符（update）

例：要将id为63346fa3dd53a05e7a9084df的book的favCount加1

db.books.update(&#123;_id:ObjectId("63346fa3dd53a05e7a9084df")&#125;,&#123;$inc:&#123;favCount:1&#125;&#125;)
//query：&#123;_id:ObjectId("63346fa3dd53a05e7a9084df")&#125;
//update：&#123;$inc:&#123;favCount:1&#125;&#125;
//输出结果为：WriteResult(&#123; "nMatched" : 1, "nUpserted" : 0, "nModified" : 1 &#125;)
  • nMatched：是否匹配（匹配1，不匹配0）；
  • nUpserted：如果没有所查找的数据，是否有插入数据（插入1，没插入0）；
  • nModified：找到查找数据后，是否有更新修改数据（有修改1，没修改0）；

若不加{multi:true}，那么只会更新第一条tag为npsql的数据，若加上，则将所查询出的数据全部更新

例：要给tag为nosql的书新增一个publishDate的属性（new Date（）为mongodb默认数据结构）

db.books.update(&#123;tag:"nosql"&#125;,&#123;$set:&#123;publishDate:new Date()&#125;&#125;)
//输出：WriteResult(&#123; "nMatched" : 1, "nUpserted" : 0, "nModified" : 1 &#125;)
db.books.update(&#123;tag:"nosql"&#125;,&#123;$set:&#123;publishDate:new Date()&#125;&#125;,&#123;multi:true&#125;)
//输出：WriteResult(&#123; "nMatched" : 9, "nUpserted" : 0, "nModified" : 9 &#125;)

其他简便命令

对于db.collection.update(query,update,options)，update可替换为：

  • updateOne：更新单个文档；
  • updateMany：更新多个文档
  • replaceOne：替换单个文档
2. upsert命令（options）
db.books.update(
... &#123;title:"my book"&#125;,
... &#123;$set:&#123;tag:"nothing",type:"nothing"&#125;&#125;,
... &#123;upsert:1&#125;
... )
//先查询是否有title为my book的书，没有就建一个（因为upsert为1）；再往my book的数据中插入tag为nothing和type为nothing的属性
db.books.find(&#123;title:"my book"&#125;)
//输出：&#123; "_id" : ObjectId("634d04ad49c7cd9e18841348"), "title" : "my book", "tag" : "nothing", "type" : "nothing" &#125;
3.实现replace

若不含任何操作符，则实现replace语义，第二个文档将完全覆盖第一个（匹配值的文档）

db.books.update(&#123;title:"my book"&#125;,&#123;justTitle:"my first book"&#125;)
//输出：WriteResult(&#123; "nMatched" : 1, "nUpserted" : 0, "nModified" : 1 &#125;)
db.books.find(&#123;justTitle:"my first book"&#125;)
//输出：&#123; "_id" : ObjectId("634d04ad49c7cd9e18841348"), "justTitle" : "my first book" &#125;
4.findAndModify命令

兼容查询修改功能，只能更新单个文档

例：将id为ObjectId(“63346fa3dd53a05e7a9084cb”)的书的favCount加1，已知原favCount=27

db.books.findAndModify(&#123;query:&#123; "_id" : ObjectId("63346fa3dd53a05e7a9084cb")&#125;,update:&#123;$inc:&#123;favCount:1&#125;&#125;&#125;)
/*输出：&#123;
    "_id" : ObjectId("63346fa3dd53a05e7a9084cb"),
    "title" : "book-19",
    "type" : "technology",
    "tag" : "developer",
    "favCount" : 6,
    "author" : "xxx19"
&#125;*/
//输出为旧值27
db.books.findAndModify(&#123;query:&#123; "_id" : ObjectId("63346fa3dd53a05e7a9084cb")&#125;,update:&#123;$inc:&#123;favCount:1&#125;&#125;,new:true&#125;)
/*输出：&#123;
    "_id" : ObjectId("63346fa3dd53a05e7a9084cb"),
    "title" : "book-19",
    "type" : "technology",
    "tag" : "developer",
    "favCount" : 8,
    "author" : "xxx19"
&#125;*/
//输出为新值28

3.4删除文档

1. remove命令

remove需配合查询条件删除，若查询条件为空文档，则删除所有文档。

db.books.remove(&#123;&#125;)   //删除所有文档
//可以但不建议，不如直接删除user集合(db.user.drop())，效率更高
db.books.remove(&#123;age:28&#125;)    //删除age为28的记录
db.books.remove(&#123;age:&#123;$lt:25&#125;&#125;)    //删除age小于25的记录

若想只删除一条数据记录，则命令通用格式为：

db.collection.remove(query,justOne)  //justOne为bool值

例：删除type为novel的首条记录

db.books.remove(&#123;type:"novel"&#125;,true)
